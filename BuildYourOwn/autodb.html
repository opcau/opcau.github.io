Autonomous Data Warehouse Cookbook

Learn how to set up your Autonomous Data Warehouse, configure and test it

Pre-requisites:

1.  SQL Developer version 18.1 or later (available here: http://www.oracle.com/technetwork/developer-tools/sql-developer/overview/index.html)

2.  A functioning Oracle Cloud Infrastructure account or trial account (preferably in one of the US regions)

Instructions:

1.  Open the Autonomous Data Warehouse console in the main (left-hand) menu:

2.  Select the “root” compartment if not already set:

 3.  Click “Create Autonomous Data Warehouse”

4.  Enter the following details in the “Create Autonomous Data Warehouse” dialog:

Display Name: ADWTest
Database Name: ADWTest
CPU Core Count: 1
Storage (TB): 1
Administrator Password: A complex password of your choosing
Confirm Password: As above

Select “Subscribe to New Database Software Licenses and the Database Cloud Service”

5.  Click “Create Autonomous Data Warehouse”

6.  When the State has transitioned to “Available” click on “ADWTest”.

7.  Click on “Service Console”:

8.  Enter the username “admin”, the password you set in step 3 above and click “Sign in”:

9.  Click on the Administration tab:

10  Click on “Download Client Credentials”

11.  Enter a password of your choosing to secure the wallet and click “Download”:

12.  Take note of the location of your wallet file for later use.

13.  Launch SQL Developer.

14.  Click the “+”

15.  Enter the following information in the dialog box:

Connection Name: admin_high
Username: admin
Password: As provided in step 4
Save Password: Selected
Connection Type: Cloud PDB
Role: Default
Configuration File: Browse and select the wallet file you saved earlier.
Keystore Password: As provided in step 11
Service: adwtest_high

16. Click “Save” and then click “Connect”

17. In the worksheet pane, enter the following (long running) SQL query:

select sum(lo_extendedprice*lo_discount) as revenue

from ssb.lineorder, ssb.dwdate

where lo_orderdate = d_datekey

and d_yearmonthnum = 199401

and lo_discount between 4 and 6

and lo_quantity between 26 and 35;

18. Click the run query button in the top pane:

19. While this query is running return to the ADW service console website and click on “Scale Up/Down” in the top menu:

20. Enter a CPU count of 4 and click “Update”

21. Notice that as the state transitions from “Scaling In Progress…” to “Available” your query continues to run – all scaling activities are completed with the database online and fully functioning.

22. Once again, click on “Scale Up/Down”

23. Enter a CPU Core Count of “1” and click on “Update”

24. Once again, your query will continue to run as the database scales itself down dynamically. This is a feature no other cloud based Data Warehouse offers, and it is completed in a matter of seconds with no more than a few clicks!

If you’d like to learn more about the Oracle Autonomous Data Warehouse – feel free to try a range of additional how-to activities available now on the Oracle GitHub: https://oracle.github.io/learning-library/workshops/journey4-adwc
